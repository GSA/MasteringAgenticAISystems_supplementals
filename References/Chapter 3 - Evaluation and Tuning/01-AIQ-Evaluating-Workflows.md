## Overview

The AIQ toolkit provides comprehensive evaluation capabilities to assess workflow performance using built-in evaluators and custom extensions. The `aiq eval` command processes datasets through configured workflows and generates performance metrics.

## Basic Evaluation Command

To evaluate a workflow, execute:
```bash
aiq eval --config_file=examples/simple/configs/eval_config.yml
```

## Configuration Structure

The evaluation configuration requires three main components:

### 1. Dataset Section
Specifies the input data source. Supported formats include:
- JSON
- JSONL
- CSV
- XLS
- Parquet

The dataset should contain questions and expected answers for testing.

### 2. Evaluators Section
Defines which metrics to apply:
- Evaluator type specification
- Metric name definition
- Associated judge LLM for scoring

### 3. Output Section
Determines where results are stored via the `output_dir` parameter

## Available Evaluators

### RAGAS Evaluator

RAGAS provides "end-to-end evaluation of RAG workflows" through three primary metrics:

#### AnswerAccuracy
- Compares generated responses against ground truth answers
- Validates correctness of the agent's output

#### ContextRelevance
- Assesses whether retrieved context matches query intent
- Evaluates retrieval quality

#### ResponseGroundedness
- Evaluates whether responses align with retrieved context
- Ensures responses are based on available information

**Configuration Requirements:**
- Judge LLM with 8-token maximum for optimal performance

**Recommended Judge Models** (ranked by effectiveness):
1. Mixtral 8x22B Instruct
2. Mixtral 8x7B Instruct
3. Llama 3.1 70B Instruct
4. Llama 3.3 70B Instruct

### Trajectory Evaluator

- Uses intermediate steps generated by the workflow
- Evaluates execution quality
- Relies on judge LLM to assess decision-making patterns
- Analyzes tool selection and reasoning paths

## Output Formats

### Workflow Output
File: `workflow_output.json`
Contains:
- Input questions
- Generated answers
- Intermediate execution steps for each dataset entry

### Evaluator Output
Metric-specific JSON files with:
- Individual scores (0-1 scale)
- Averaged performance metrics across all dataset entries
- Reasoning explanations

## Remote Workflow Evaluation

Evaluate workflows running on remote servers:

1. **Start the remote AIQ server:**
   ```bash
   aiq serve --config_file=config.yml
   ```

2. **Run local evaluation with endpoint flag:**
   ```bash
   aiq eval --config_file=eval_config.yml \
     --endpoint http://localhost:8000
   ```

## Configuration Overrides

Command-line overrides allow runtime parameter adjustment:
```bash
aiq eval --config_file eval_config.yml \
  --override llms.nim_rag_eval_llm.temperature 0.7
```

## Viewing Available Evaluators

List all registered evaluators:
```bash
aiq info components -t evaluator
```

## Advanced Features

### REST API
Access evaluation functionality through the `/evaluate` endpoint

### Custom Evaluators
Extend capabilities by implementing and registering custom evaluation logic

### Profiling Integration
Monitor workflow performance metrics during evaluation

## Best Practices

1. **Start with RAGAS** for RAG-based workflows
2. **Use Trajectory Evaluator** for agent decision-making assessment
3. **Compare Judge Models** to find best fit for your domain
4. **Iterate on Metrics** based on your specific use case
5. **Monitor Performance** across evaluation runs to track improvements
